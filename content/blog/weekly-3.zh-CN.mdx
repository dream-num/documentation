## 概览

本周我们隆重推出了全新的 Univer 文档，并正式发布了 **Univer v0.9.1**。此次版本更新聚焦于自定义组件的注册与使用，进一步提升了 Univer 的扩展能力和开发体验。

## 文档更新

我们对 Univer 官方文档进行了全新设计，致力于为开发者提供更清晰、结构更合理的文档体验。新版文档在内容编排、检索效率以及可读性方面均有显著提升。此外，我们计划在后续持续补充丰富的在线演示和示例代码，帮助开发者更高效地上手与集成 Univer。欢迎大家持续关注文档的更新进展，并反馈宝贵意见。

## Univer v0.9.1

### UI 适配器体系

从 v0.9.1 开始，Univer 支持将 **Vue 3.x、Vue 2.x 以及 Web Components** 等多种前端框架的组件注册为自定义组件。这意味着开发者可以根据自身项目技术栈灵活选择组件开发框架，极大提升了兼容性和扩展性。

为实现这一目标，我们对 `@univerjs/ui` 包进行了架构调整，移除了对 Vue 3.x 的内置适配。开发者如需集成 Vue 3.x 组件，需要单独安装 `@univerjs/ui-adapter-vue3` 包，并通过注册 `UniverVue3AdapterPlugin` 插件来完成适配。这样不仅避免了对特定 Vue 版本的依赖，也为未来支持更多框架打下了基础。

#### 使用示例

**预设模式**

```typescript
createUniver({
  presets: [
    UniverSheetsCorePreset(),
  ],
  plugins: [
    UniverVue3AdapterPlugin, // [!code: ++]
  ],
})
```

**插件模式**

```typescript
univer.registerPlugin(UniverUIPlugin)

// UI 适配器插件必须在 UniverUIPlugin 之后注册
univer.registerPlugin(UniverVue3AdapterPlugin) // [!code: ++]
```

此外，我们还发布了 `@univerjs/ui-adapter-web-component` 包，使开发者可以通过 Web Components 进行自定义组件开发。对于使用 Angular 的团队，可以结合 `@angular/elements` 实现组件适配，进一步拓宽了技术选型空间。

### 新增预设包

本次更新新增了 `@univerjs/preset-docs-node-core` 预设包。通过该包，开发者能够在 Node.js 环境下操作 Univer Docs 的数据，为后端文档处理、自动化脚本及服务端渲染等场景提供了便利。

### 其他功能与修复

- **自定义行列标题 API 优化**
  - 新增工作簿级别的行列标题自定义支持：`FWorkbook.customizeRowHeader` 和 `FWorkbook.customizeColumnHeader`
  - 修复了工作表级别 API（`FWorksheet.customizeRowHeader` 和 `FWorksheet.customizeColumnHeader`）的相关问题
- **兼容性与体验优化**
  - 解决了在 webpack@5 环境下使用 `@univerjs/preset-sheets-advanced` 预设时出现白屏的问题
  - 修复了 Univer 全局 CSS 在 Tailwind CSS v4 环境下的样式冲突
  - 修复了非 React 自定义组件集成时的警告提示问题
  - 解决了在未使用 Web Worker 进行公式计算时，自动填充手柄按钮异常消失的 bug
  - 修复了点击 fx 按钮进入公式编辑器时无法用 BackSpace 删除公式的问题
  - 优化了重新进入公式编辑模式时的焦点行为，解决了公式介绍弹窗显示异常等细节问题
