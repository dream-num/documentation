欢迎来到 **Univer 周报** 的第二期！在这个系列中，我们将每周为您带来 Univer 的最新动态，分享项目的进展、亮点和更新。希望这些内容能够帮助您更深入地了解并使用 Univer，同时欢迎您积极参与我们的开发与成长！

## **Univer Go 正式发布**

我们非常高兴地向大家介绍 [Univer Go](https://go.univer.ai)！这是一款全新的桌面应用，您可以借助它创建个性化电子表格，并与 AI 代理、数据库及各种应用实现无缝集成。

## **Univer SDK 0.5.3 版本上线**

Univer SDK 的 OSS 和 Pro 版本迎来了全新的 0.5.3 更新！在此次版本中，我们对许多 Facade API 进行了优化，使链式调用成为可能，从而让电子表格的操作更加便捷。需要注意的是，此次更新引入了一些破坏性变更，请参考我们的[发布日志](https://www.univer.ai/zh-CN/changelog)了解如何适配这些改动。此外，我们还新增了向电子表格添加“范围主题”的功能，进一步增强了使用体验。该功能目前仅支持通过 Facade API 调用。

有关更多详情，请查阅发布日志。

## **React 19 兼容性**

我们仍在致力于支持 React 19.0.0 的兼容性。当前的主要阻碍在于一个依赖项与 React 19 不兼容。为了解决这一问题，我们正在开发自研组件以取代该依赖项。预计将在下周完成此工作，欢迎订阅我们的 [GitHub Issue](https://github.com/dream-num/univer/pull/4247) 以获取最新进展。

## **协同功能优化**

我们已着手优化 Univer 的协同功能。本周的重点是将“快照保存”和“历史记录生成”过程异步化。这项改进将显著提升协同引擎的吞吐量，更多精彩内容请关注下期 Univer 周报！

## **新博客上线**

我们发布了一篇新博客，详细讲述了 Facade API 的构思、实现及优化过程，并阐述了它如何成为 Univer Go 项目的核心支柱。点击[这里](/zh-CN/blog/facade)阅读完整文章。
