## 從範本建立

若您希望在本機環境建立 Univer 應用，可執行以下指令，透過範本建立一個新的專案。

<Tabs items={['npm', 'pnpm']}>
  <Tab>
    執行下列指令前請確認環境已安裝 `nodejs >= 18`、`npm >= 8` 與 `git`。

    ```shell
    npx degit dream-num/univer-sheet-start-kit univer-sheet-start-kit
    cd univer-sheet-start-kit
    npm install
    npm run dev
    ```
  </Tab>
  <Tab>
    執行下列指令前請確認環境已安裝 `nodejs >= 18`、`pnpm >= 7` 與 `git`。

    ```shell
    pnpm dlx degit dream-num/univer-sheet-start-kit univer-sheet-start-kit
    cd univer-sheet-start-kit
    pnpm install
    pnpm dev
    ```
  </Tab>
</Tabs>

至此，您已成功建立第一個屬於您的 Univer 專案，並體驗了部分 Univer 功能。

## 基本概念 [#basic-concepts]

在進一步深入了解 Univer 前，請先閱讀本節以掌握其基本概念，有助您更順利地使用 Univer。

基本概念包含：

- 插件（Plugin）
- 快照（Snapshot）
- 命令（Command）
- Facade API

### 插件

Univer 的功能皆由插件提供；插件是 Univer 的基本組成單元，多個插件組合即構成一個 Univer 應用。

插件的職責包含：

1. 實作具體功能，例如 `@univerjs/sheets-filter` 提供篩選功能
2. 提供底層能力，例如 `@univerjs/engine-render` 提供渲染能力

您可依需求挑選並組合 Univer 所提供的插件，客製出符合情境的應用，避免載入不必要的程式碼。您亦可撰寫自訂插件以擴充功能，滿足個別化需求。

### 快照

在 Univer 中需載入「快照」才能載入文件。每種文件類型都有其專屬的快照格式，請參考下列 API 文件以撰寫符合格式的快照：

1. 試算表的快照格式為 [`IWorkbookData`](/guides/sheets/model/workbook-data#iworkbookdata)
2. 文件的快照格式為 [`IDocumentData`](https://reference.univer.ai/zh-CN/interfaces/IDocumentData)

功能插件可透過 `resources` 欄位儲存資源（如圖片等），可參考[插件自訂模型](/guides/recipes/tutorials/custom-model)。

<Callout type="warning" title="注意事項">
  注意！Univer 的快照僅用於儲存資料；在執行階段它不會自動反映最新的文件狀態。若您想將當前文件狀態保存為快照，請使用 Facade API 所提供的保存快照方法。
</Callout>

### 命令

在 Univer 中，所有對資料的修改皆須透過「命令」執行。命令導向的模式可更好追蹤變更，並支援復原、重做、協同編輯與處理多功能間的複雜關聯邏輯。

若您想以命令方式修改文件資料，請參考[如何查找命令 ID](/guides/recipes/practices/find-the-command-id)。若您改用 Facade API 修改，底層其實是由 Facade API 代您組裝參數並執行對應命令。

<Callout type="warning" title="注意事項">
  **請勿直接修改 snapshot！** 這樣不會生效，亦不會觸發視圖或 UI 更新；請透過命令或 Facade API 進行資料修改。
</Callout>

### Facade API

由於辦公類應用的複雜度極高，Univer 的架構也相對複雜。為降低使用門檻，我們提供 Facade API——這是對 Univer 官方插件的封裝，提供更簡潔的介面，協助開發者快速上手。

目前 Facade API 仍在持續完善中，若您有任何建議或需求，歡迎於 GitHub 提出 issue。

當然，簡化的同時亦代表取捨；若您需要更進階的自訂能力，則需在理解 Univer 架構後，直接使用較底層的命令與模組所曝露的 API。
