## スターターキット

ローカル開発環境で始めたい場合は、以下のスターターキットを使用して、数秒で新しい Univer アプリを作成できます。

<Tabs items={['npm', 'pnpm']}>
  <Tab>
    以下のコマンドを実行するには、デバイスに `nodejs >= 18`、`npm >= 8`、および `git` がインストールされている必要があります。

    ```shell
    npx degit dream-num/univer-sheet-start-kit univer-sheet-start-kit
    cd univer-sheet-start-kit
    npm install
    npm run dev
    ```
  </Tab>
  <Tab>
    以下のコマンドを実行するには、デバイスに `nodejs >= 18`、`pnpm >= 7`、および `git` がインストールされている必要があります。

    ```shell
    pnpm dlx degit dream-num/univer-sheet-start-kit univer-sheet-start-kit
    cd univer-sheet-start-kit
    pnpm install
    pnpm dev
    ```
  </Tab>
</Tabs>

素晴らしい仕事です！Univer の最初のステップを成功裏にクリアし、Univer の機能を体験しました。

## 基本概念 [#basic-concepts]

Univer の深い理解を得るために、まずはこのドキュメントを読んで、Univer の基本概念を理解してください。これにより、Univer をよりスムーズに使用できるようになります。

基本概念には以下が含まれます。

- プラグイン
- スナップショット
- コマンド
- Facade API

### プラグイン

Univer のすべての機能はプラグインによって提供されており、プラグインは Univer の基本的な構成要素です。複数のプラグインを組み合わせることで、Univer アプリケーションが形成されます。

プラグインの責任には以下が含まれます。

1. 機能の実装、例えば `@univerjs/sheets-filter` はフィルタリング機能を提供します。
2. 基本的な機能の提供、例えば `@univerjs/engine-render` はレンダリング機能を提供します。

Univer のプラグインを組み合わせて、要件に合った Univer アプリケーションをカスタマイズし、不要なコードの読み込みを回避できます。また、プラグインを作成して Univer の機能を拡張し、個別のニーズに対応することもできます。

### スナップショット

Univer でドキュメントを読み込むには、スナップショットを読み込む必要があります。各種のドキュメントはそれぞれ独自のスナップショット形式を提供します。形式要件を満たすスナップショットの書き方については、上記の API ドキュメントを参照してください。

1. スプレッドシートのスナップショット形式は [`IWorkbookData`](/guides/sheets/model/workbook-data#iworkbookdata) です。
2. ドキュメントのスナップショット形式は [`IDocumentData`](https://reference.univer.ai/en-US/interfaces/IDocumentData) です。

機能プラグインは、`resources` フィールドを通じて画像などのリソースを保存できます。詳細については、[プラグインカスタムモデル](/guides/recipes/tutorials/custom-model)を参照してください。

<Callout type="warning" title="注意事項">
  注意! Univer のスナップショットはデータを保存するためだけに使用されます。実行時の最新のドキュメント状態を反映するものではありません! ドキュメントの状態をスナップショットとして保存したい場合は、Facade API でスナップショットを保存するメソッドを使用してください。
</Callout>

### コマンド

Univer では、すべてのデータ変更はコマンドを通じて実行する必要があります。コマンドベースのアプローチは、値の変更をより適切に追跡し、元に戻す、やり直す、共同編集などの機能を実装し、関数間の複雑な関連ロジックを処理することができます。

もしコマンドを通じてドキュメントデータを変更したい場合は、[コマンド ID の見つけ方](/guides/recipes/practices/find-the-command-id)を参照してください。Facade API を通じてドキュメントデータを変更する場合、Facade API は実際にパラメータを準備し、背後でコマンドを実行します。

<Callout type="warning" title="注意事項">
  **スナップショットを直接変更しないでください!** これは効果がなく、ビューや UI の更新をトリガーしません。データはコマンドまたはFacade API を通じて変更してください。
</Callout>

### Facade API

オフィスアプリケーションの複雑さにより、Univer のアーキテクチャも非常に複雑です。開発者が Univer をより簡単に使用できるようにするために、Facade API を提供しています。これは Univer の第一党プラグインのラッパーであり、開発者が Univer をより簡単に使用できるようにするシンプルなインターフェースを提供します。

Facade API は改善を続けています。最新の更新にご期待ください。質問や要件がある場合は、GitHub で issue を提出してください。

もちろん、すべてには代償があります。Facade API は高レベルの API であり、基礎となる詳細を抽象化しています。複雑な機能を実装する必要がある場合は、Univer のアーキテクチャを理解し、低レベルの API を直接使用する必要があるかもしれません。
